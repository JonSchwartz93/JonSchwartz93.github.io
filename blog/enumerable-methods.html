Ruby Enumerable Methods - '.partition'

Sunday, January 10th, 2016

This week at DBC we continued to expand our understanding of Ruby by building our own methods to achieve a variety of tasks. One area of focus was the Enumerable module (a module is essentially a container of methods). Enumerable is extremely versatile and particularly useful when building methods because it equips a user with a variety of ways to search and sort through a data structure (aka - an array or hash.) The Enumerable module contains numerous useful methods, but one that I find particularly interesting is '.partition'.

Here is the description of '.partition' found in Ruby Docs:

Ruby Docs is often quite difficult to decipher, so let me break this down for you (for more information about reading Ruby Docs, click here.) When '.partition' is called on a data structure, it returns an array comprised of two arrays - one array containing the elements for which the block of code evaluates to true, and the second array containing those elements for which the block returns false. Using the example above, '.partition' is called on a range of numbers, 1-6. The block of code is evaluating which numbers in that range are even, and thus returns one array ([2, 4, 6] = the "true" array), and another array containing the remaining numbers ([1, 3, 5] = the "false" array).
